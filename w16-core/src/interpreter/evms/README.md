# Experimental Virtual Machines

В этой папке находятся экспериментальные виртуальные машины W16.

EVM здесь означает **Experimental Virtual Machine**, а модуль `evms`
расшифровывается как **Experimental Virtual Machines**.

## Orca EVM

`orca_evm.rs` содержит Orca EVM — быстрый unchecked-интерпретатор W16 bytecode.

Главная идея Orca:

```text
меньше проверок -> меньше overhead -> больше скорость
```

Orca не является заменой обычной VM. Обычная VM остаётся эталонной и более
безопасной реализацией. Orca предназначена для бенчмарков и доверенного
bytecode, который уже прошёл verifier или был сгенерирован компилятором W16.

## Что оптимизировано

Текущая Orca делает несколько агрессивных вещей:

- хранит `pc` как `*const Instruction`, а не как числовой `ip`;
- не проверяет `pc < code_len` на каждой инструкции;
- завершает выполнение только через `Halt` или `Ret` из entry-функции;
- для переходов сразу строит `pc` через `code_base.add(target)`;
- хранит return address как raw pointer;
- использует raw pointer доступ к регистрам и памяти;
- использует unchecked чтение constant pool;
- держит call stack в заранее выделенном `MaybeUninit` buffer;
- не вызывает `Vec::push` / `Vec::pop` в горячем пути вызовов;
- выносит `Print*` в cold-функции, чтобы не раздувать горячий dispatch;
- не возвращает `Result` и не содержит error path.

Это делает Orca быстрее на доверенных программах, но заметно ужесточает
контракт безопасности.

## Контракт безопасности

Перед запуском Orca caller должен гарантировать:

- `bytecode.instructions` не пустой;
- все переходы ведут на существующие инструкции;
- все обращения к памяти находятся внутри выделенной памяти;
- все обращения к constant pool валидны;
- строки для `PrintStr` являются валидным UTF-8;
- деления на ноль нет;
- signed division/rem не выполняет `i64::MIN / -1` и `i64::MIN % -1`;
- глубина вызовов не превышает `call_capacity`;
- программа завершается через `Halt` или `Ret`;
- bytecode не содержит семантически неправильных инструкций.

Если эти условия нарушены, Orca может вернуть неправильный результат, зависнуть,
вызвать panic или привести к undefined behavior.

## Когда использовать

Используйте Orca:

- для бенчмарков;
- для экспериментов с dispatch;
- для сравнения с обычной VM и JIT;
- для запуска доверенного bytecode.

Не используйте Orca:

- как публичный безопасный runtime;
- для непроверенного bytecode;
- как эталон корректности;
- вместо тестов обычной VM.
