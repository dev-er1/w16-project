# Участие в разработке W16

Прежде всего, **спасибо за ваш интерес к W16**! Вклад разработчиков помогает развивать рантайм, делая его намного лучше.

Но, прежде чем начать делать Pull requests в проект, для начала прочитайте условия снизу.

## Условия контрибьютинга
1. **Убедитесь что у вас совместимая версия**:
    * `rustc` **1.93.1**
    * `cargo` **1.93.1**

2. **Перед отправкой Pull request-а ваша версия проекта должна пройти все тесты**. Запустите тесты в корневой папке workspace-а:
    ```bash
    cargo test
    ```

3. **Оформление коммитов**. Пожалуйста, используйте понятные префиксы в сообщениях коммитов,
чтобы было ясно, какую часть рантайма затронули изменения:
```text
feat(...):     ... — добавление новой фичи в какой то части проекта.
fix(...):      ... — исправление.
doc(...):      ... — добавление или исправление документации.
chore(...):     ... — добавление или исправление некоторых вещей в проекте(маленькие изменения).
refactor(...): ... — изменение логики, или структуры проекта.
```

## Как развернуть проект локально
```bash
# Используйте HTTPS / SSH на ваш выбор

# Через HTTPS
git clone https://github.com/dev-er1/w16-project.git

# Через SSH
git clone git@github.com:dev-er1/w16-project.git

cd w16-project
```

## Условия по коду и стилю написания кода
- **Комментарии**: желательно комментарии писать на русском, но вы можете писать на любом языке.
- **Стиль написания кода**: пишите как хотите.
